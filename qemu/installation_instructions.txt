1) qemu под винду лежит тут: http://www1.interq.or.jp/t-takeda/qemu/qemu-0.9.1-windows.zip
Распаковать в Program Files, например.
Туда же положить образ лайвсиди, чтоб не писать длинные строчки

2) зайти туда, запустить cmd, написать 
qemu-img.exe create myimage.img 100M

положить туда файлик launch.cmd, который лежит рядом с этим в репозитории, запустить его

3) На экране образуется хуйня. Бесстрашно жмём tab, space, "single" (без кавычек), enter
Идём пить чай.

4) там пишем cfdisk, внутри всё довольно интуитивно, создаём партицею, write, выходим.

mke2fs /dev/hda1

mount /dev/hda1

cd /media/hda1/

mkdir icfpc 

cd icfpc

wget http://smlnj.org/icfp08-contest/lo-res.tgz
wget http://smlnj.org/icfp08-contest/sample-maps.tgz

man tar, там сразу будет дичайшая последовательность букв для распаковки сразу из tgz
копируем server в папку к картам

shutdown now (это команда. Было бы обидно проебать всё нажитое непосильным трудом из-за
того, что у хуюникса хрупкая файловая система).

5) закрываем qemu и запускаем ещё раз. Идём пить чай. Надолго.

6) когда загрузится говнокде находим снизу консольку и тыкаем мышкой один раз. 
Один раз. Не даблкликом, просто один раз. Ждём полминуты, опять маунтим hda1, 
ползём к распакованному серверу, пишем
./server -v имякарты

Всё, теперь можно коннектиться клиентом с реального компа к 127.0.0.1 по ихейному 17676 порту.

Сервер у них глючной кстати сейчас и нихуя не рвёт соединение.
